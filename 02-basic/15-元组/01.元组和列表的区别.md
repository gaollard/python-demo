# 元组和列表的区别

元组（tuple）和列表（list）都是有序序列，都能用下标访问、切片、遍历。核心差别：**列表可变，元组不可变**。

## 对比一览

| | 列表 `list` | 元组 `tuple` |
|---|---|---|
| 写法 | `[1, 2, 3]` | `(1, 2, 3)` 或 `1, 2, 3` |
| 可变性 | 可变（mutable） | 不可变（immutable） |
| 增删改 | `append` / `pop` / 下标赋值 | 不支持 |
| 性能 | 稍慢 | 稍快、更省内存 |
| 可哈希 | 否（不能做 dict 的键） | 元素都可哈希时可以 |
| 典型用途 | 同质、会变化的数据集合 | 固定结构、当作记录/返回多值 |

## 可变 vs 不可变

```python
# 列表可以就地修改
nums = [1, 2, 3]
nums[0] = 10
nums.append(4)
print(nums)  # [10, 2, 3, 4]

# 元组不能修改元素
dims = (200, 50)
# dims[0] = 250  # TypeError: 'tuple' object does not support item assignment
```

注意：元组本身不可变，但若元素是可变对象，该对象内部仍可改：

```python
t = ([1, 2], 3)
t[0].append(99)
print(t)  # ([1, 2, 99], 3)
# t[0] = [1, 2, 99] 仍然会报错 —— 换引用不行，改内部可以
```

## 定义方式

```python
a = (1, 2, 3)
b = 1, 2, 3          # 括号可省略
c = (1,)             # 单元素元组必须有逗号
d = (1)              # 这是整数 1，不是元组
e = ()               # 空元组
f = tuple([1, 2, 3]) # 从列表转换
```

## 重新赋值 ≠ 修改

元组不能改内容，但变量可以指向一个新元组：

```python
dimensions = (200, 50)
dimensions = (400, 100)  # 合法：变量换了引用，旧元组未变
print(dimensions)        # (400, 100)
```

## 何时用元组

1. **数据不应被修改**（配置、坐标、RGB）
2. **函数返回多个值**（本质返回一个元组）
3. **需要做字典的键或放入集合**（列表不行）
4. **解包赋值**更语义清晰：`name, age = ("Alice", 25)`

```python
# 做字典的键
locations = {
    (0, 0): "原点",
    (1, 2): "点 A",
}

# 函数返回多值
def get_size():
    return 1920, 1080  # 返回元组

width, height = get_size()
```

## 常用操作（与列表相同的部分）

```python
t = (10, 20, 30, 40)

print(t[0])       # 10
print(t[-1])      # 40
print(t[1:3])     # (20, 30)
print(len(t))     # 4
print(20 in t)    # True

for x in t:
    print(x)

# 只有 count / index，没有 append、remove 等修改方法
print(t.count(20))  # 1
print(t.index(30))  # 2
```

## 列表 ↔ 元组

```python
lst = [1, 2, 3]
tup = tuple(lst)   # (1, 2, 3)
back = list(tup)   # [1, 2, 3]
```

## 小结

- 需要**增删改** → 用列表
- 需要**固定、安全、可哈希** → 用元组
- 两者都是序列，索引 / 切片 / 遍历写法几乎一样
