# 模块内置变量

一个 `.py` 文件就是一个模块。导入后，Python 会给模块对象挂上一批**双下划线（dunder）属性**，用来描述模块自身信息。

> 注意：这里说的是「模块对象上的内置属性」，不是标准库里的内置模块（见 [12.内置模块有哪些](./12.内置模块有哪些.md)）。

---

## 常用内置属性

| 属性 | 含义 |
|------|------|
| `__name__` | 模块名；直接运行时为 `"__main__"`，被 `import` 时为模块名 |
| `__file__` | 模块源文件的路径 |
| `__doc__` | 模块文档字符串（文件顶部的三引号字符串；没有则为 `None`） |
| `__package__` | 所属包名；顶层模块多为 `None` |
| `__loader__` | 加载该模块的 loader |
| `__spec__` | 模块规格对象（`ModuleSpec`） |
| `__cached__` | 对应的 `.pyc` 缓存路径 |
| `__builtins__` | 内置命名空间（dict 或 `builtins` 模块） |
| `__dict__` | 模块的命名空间字典（模块里定义的变量/函数/类都在这里） |
| `__annotations__` | 模块级类型注解（若有） |

---

## `__name__` 最常用

```python
# demo.py
print(__name__)

if __name__ == "__main__":
    print("直接运行本文件")
else:
    print("被其他文件 import")
```

```bash
python demo.py          # 输出：__main__
                        # 直接运行本文件

python -c "import demo" # 输出：demo
                        # 被其他文件 import
```

常见写法：把「可执行入口」放在 `if __name__ == "__main__":` 里，这样被导入时不会误执行。

---

## 快速查看模块有哪些内置变量

```python
# 在任意 .py 文件中
print("__name__:", __name__)
print("__file__:", __file__)
print("__doc__:", __doc__)
print("__package__:", __package__)
print("__cached__:", __cached__)

# 列出当前命名空间里所有双下划线开头的名字
print([k for k in dir() if k.startswith("__")])
```

也可以对已导入的模块查看：

```python
import math

print(math.__name__)    # math
print(math.__file__)    # 标准库 math 的路径
print(math.__doc__)     # 文档字符串开头一段
print(dir(math))        # 模块里有什么（含函数、常量等）
```

## `__dict__`：模块的命名空间

模块里定义的名字，本质都存在 `__dict__` 里：

```python
# m.py
value = 0

def update_val():
    global value
    value = 1
```

```python
import m

print(m.value)           # 0
print(m.__dict__["value"])  # 0，等价访问

m.update_val()
print(m.value)           # 1
print(m.__dict__["value"])  # 1
```

这也解释了：`from m import value` 拿到的是**当时的值拷贝**；之后改 `m.value`，已绑定的 `value` 不会跟着变。要始终看最新值，应访问 `m.value`。

---

## 小结

1. 每个模块都有一组 dunder 属性，描述「我是谁、文件在哪、文档是什么」。  
2. 日常最重要的是 `__name__`（区分直接运行 / 被导入）和 `__file__`（定位路径）。  
3. 模块的全局变量存在 `__dict__` 中；`from ... import` 是按值绑定，不是持续引用模块属性。
