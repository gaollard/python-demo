# 模块 `__all__`

![20260723220523](http://s3.airtlab.com/blog/20260723220523.png)

`__all__` 是模块级特殊变量，用来指定 `from 模块名 import *` 时**哪些名字会被导入**（通配匹配哪些功能）。

---

## 作用

| 情况 | `from m import *` 导入什么 |
|------|---------------------------|
| 定义了 `__all__` | 只导入列表里列出的名字 |
| 未定义 `__all__` | 导入不以 `_` 开头的公开名字 |

注意：`__all__` **只影响** `import *`。显式导入不受限制：

```python
from my import NAME          # 可以，即使 NAME 不在 __all__ 里
from my import log_separator2  # 也可以
```

---

## 示例

```python
# my.py
__all__ = ["log_separator1", "log_separator3", "PI"]

PI = 3.1415926
NAME = "黑马☆涛哥"

def log_separator1():
    print("-" * 50)

def log_separator2():
    print("*" * 50)

def log_separator3():
    print("=" * 50)

def log_separator4():
    print("#" * 50)
```

```python
# test.py
from my import *

log_separator1()   # OK
log_separator3()   # OK
print(PI)          # OK → 3.1415926

# 下面这些不在 __all__ 中，通配导入后不可用 → NameError
# log_separator2()
# log_separator4()
# print(NAME)
```

---

## 未定义 `__all__` 时

```python
# utils.py（没有 __all__）
public_val = 1
_private_val = 2   # 以下划线开头

def foo():
    pass

def _bar():
    pass
```

```python
from utils import *

print(public_val)  # OK
foo()              # OK
# _private_val / _bar 不会被 * 导入
```

约定：以下划线开头表示「内部用」；没有 `__all__` 时，`import *` 会自动跳过它们。

---

## 包里的 `__all__`

在包的 `__init__.py` 里也可以定义 `__all__`，控制 `from package import *` 暴露哪些子模块或名字：

```python
# mypkg/__init__.py
from .app import MiniHTTP
from .request import Request

__all__ = ["MiniHTTP", "Request"]
```

这样既声明了公开 API，也方便通配导入时只拿到对外接口。

---

## 小结

1. `__all__` 只约束 `from ... import *`，不阻止 `from ... import 具体名字`。  
2. 有 `__all__` → 只导入列表中的名字；没有 → 导入非 `_` 开头的名字。  
3. 适合用来声明模块 / 包的**公开 API**，避免把内部函数、常量一并暴露出去。
