# 软件包 Package

一个 `.py` 文件是**模块（module）**；把多个模块按目录组织起来，就是**包（package）**。

包让代码可以分层：`core.logger`、`utils.string`，而不是一堆平铺的 `.py`。

---

## 模块 vs 包

| | 模块 | 包 |
|--|------|----|
| 形态 | 单个 `.py` 文件 | 一个目录（通常含 `__init__.py`） |
| 导入名 | `import circle_func` | `import core.logger` |
| 作用 | 放一组相关功能 | 把多组相关模块归到一起 |

```text
01module1/
├── circle_func.py      # 模块
├── main.py
├── test.py
└── core/               # 包
    ├── __init__.py     # 包标识（可为空）
    └── logger.py       # 包内模块
```

---

## `__init__.py` 的作用

目录要能被当成包导入，传统写法是放一个 `__init__.py`（可以为空）。

导入包时会发生两件事：

1. 把该目录识别为包  
2. **执行** `__init__.py` 里的代码（适合做包级初始化、统一导出）

```python
# core/__init__.py
from .logger import log, info, warning, error, critical

__all__ = ["log", "info", "warning", "error", "critical"]
```

之后可以：

```python
import core
core.log("test")

# 或
from core import log
log("test")
```

没有在 `__init__.py` 里导出时，一般要写完整路径：

```python
from core.logger import log
# 或
import core.logger as logger
logger.log("test")
```

> Python 3.3+ 支持**命名空间包**（没有 `__init__.py` 也能导入），但项目里仍建议显式写上，语义更清晰。

---

## 导入方式

以 `core/logger.py` 为例：

```python
# 1. 导入包内模块，起别名
import core.logger as logger
logger.log("test")

# 2. 从包内模块导入具体名字
from core.logger import log, info, warning, error, critical
log("test")

# 3. 若 __init__.py 已 re-export，可从包直接拿
from core import log
```

对应示例见：`02-basic/模块/01module1/test.py`。

## 相对导入

包内部互相引用时，可用相对导入（以 `.` 表示当前包）：

```python
# core/app.py
from .logger import log          # 同级模块
from .utils.helpers import foo   # 子包模块
from ..other import bar          # 上级包（少用，容易绕晕）
```

注意：

- 相对导入只在**包内**有效  
- 直接 `python core/app.py` 跑文件时，相对导入常会失败  
- 推荐用 `python -m` 以模块方式运行（见 [01.python-m-的作用](./01.python-m-的作用.md)）

```bash
# 在项目根（能找到包的那一层）执行
python -m core.app
```

## 包的内置属性

包也是模块对象，同样有 dunder 属性（见 [13.模块内置变量](./13.模块内置变量.md)）：

| 属性 | 在包里常见含义 |
|------|----------------|
| `__name__` | 包名，如 `"core"`、`"core.logger"` |
| `__package__` | 所属包；包自身常等于 `__name__` |
| `__path__` | **包独有**：搜索子模块的目录列表 |
| `__file__` | 通常是 `__init__.py` 的路径 |

```python
import core

print(core.__name__)   # core
print(core.__path__)   # ['.../core']
print(core.__file__)   # '.../core/__init__.py'
```

---

## 包里的 `__all__`

在 `__init__.py` 里定义 `__all__`，控制 `from package import *` 暴露什么（详见 [14.模块__all__](./14.模块__all__.md)）：

```python
# core/__init__.py
from .logger import log, info

__all__ = ["log", "info"]
```

```python
from core import *
log("ok")
# warning 不在 __all__ 里，通配导入后不可用
```

---

## 小结

1. **模块** = 一个 `.py`；**包** = 含模块的目录（通常有 `__init__.py`）。  
2. `__init__.py` 标识包，并在 `import 包` 时执行，适合统一导出公开 API。  
3. 包内优先用 `from .xxx import ...`；对外用 `from core.logger import ...`。  
4. 直接运行文件容易踩相对导入坑，包内入口用 `python -m`。  
5. 包也有 `__name__` / `__path__` 等属性；`__all__` 可声明包的公开接口。
