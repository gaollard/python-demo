`global` 是 Python 的关键字（不是函数），用来在函数内部声明：接下来对某个名字的赋值，作用在**模块级全局变量**上，而不是新建一个局部变量。

核心要记住：函数里**读取**全局变量通常不需要 `global`；只有在函数里**给全局变量重新赋值**时，才需要 `global`。

📌 先搞清：读全局 vs 写全局

读全局（不需要 global）

num = 10

def test():
    print(num)  # 直接读全局 num → 10

test()

写全局（需要 global）

num = 10

def test():
    num = 20      # ❌ 没有 global：这里创建的是局部变量，全局 num 不变
    print(num)    # 20

test()
print(num)        # 还是 10

正确写法：

num = 10

def test():
    global num
    num = 20      # ✅ 修改的是全局 num
    print(num)    # 20

test()
print(num)        # 30（若再赋 30 同理）

对照你练习里的例子：

num = 10

def test():
    num = 20
    print(num)  # 20（局部）

test()
print(num)      # 10（全局未变）

def test2():
    global num
    num = 30
    print(num)  # 30

test2()
print(num)      # 30（全局已改）

🔍 为什么需要 global？

Python 用「赋值即定义」判断作用域：

- 函数里只要出现对某个名字的赋值，解释器默认把它当成**局部变量**
- 若该局部变量在赋值前就被读取，会报错：`UnboundLocalError`

x = 1

def f():
    print(x)   # ❌ UnboundLocalError：下面有赋值，x 被当成局部，但此时还没赋值
    x = 2

f()

解决办法二选一：

1. 用 `global` 明确说「我改的是全局」
2. 不要在函数里给同名全局变量赋值（推荐：返回新值，由调用方决定是否更新）

⚖️ global 能做什么、不能做什么

能做：

# 修改已有全局变量
count = 0

def add():
    global count
    count += 1

# 在函数里「创建」全局变量（模块里原先没有也可以）
def init():
    global name
    name = 'Alice'

init()
print(name)  # Alice

不能做 / 容易误解：

1. `global` 不是「让变量到处可见」，它只管**当前模块**的模块级名字
2. 别的模块要改这个变量，仍要通过 `import` 拿到模块对象再改，或对方模块自己用 `global`
3. `global` 只影响**名字绑定（赋值）**；改可变对象内部内容，通常不需要 `global`

# 可变对象：改内容不需要 global
nums = [1, 2, 3]

def append_one():
    nums.append(4)   # ✅ 改的是列表对象本身，不是重新绑定名字

append_one()
print(nums)  # [1, 2, 3, 4]

def replace_list():
    global nums
    nums = [9, 9]    # ✅ 重新绑定名字才需要 global

replace_list()
print(nums)  # [9, 9]

⚠️ 常见陷阱

1. 漏写 global，以为改了全局

total = 0

def add(n):
    total = total + n   # ❌ UnboundLocalError 或只改局部

# 正确：
def add(n):
    global total
    total = total + n

2. 多个名字要一起声明

a = 1
b = 2

def update():
    global a, b
    a = 10
    b = 20

3. global 声明要写在函数里、使用该名字之前

def bad():
    x = 1
    global x   # ❌ SyntaxError：global 必须在赋值之前

4. 能不用就不用

过度使用 `global` 会让状态分散、难测、难读。更常见的写法是：

def add(total, n):
    return total + n

total = 0
total = add(total, 1)

或把状态放进类 / 字典等显式容器里。

📦 和 nonlocal 的区别（提前一眼）

| 关键字     | 作用对象                         | 典型场景           |
| ---------- | -------------------------------- | ------------------ |
| `global`   | 模块级全局变量                   | 函数改模块变量     |
| `nonlocal` | 外层函数的局部变量（非全局）     | 闭包改外层局部变量 |

def outer():
    n = 0
    def inner():
        nonlocal n
        n += 1
    inner()
    return n

📌 一句话总结

函数内**读**全局变量一般不用 `global`；函数内**赋值/重新绑定**全局名字时才用 `global`。改列表、字典等内容不必 `global`，换掉整个对象才需要。能返回值或传参解决时，尽量少用全局可变状态。
