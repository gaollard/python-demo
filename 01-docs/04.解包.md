Python 中的“解包”（Unpacking）是指将可迭代对象（如列表、元组、字典）中的元素自动提取出来，并分别赋值给多个变量的过程。它能让代码摆脱繁琐的索引访问，变得极其简洁优雅。

📦 基础解包

最基础的解包要求左侧变量数量必须与右侧元素数量严格相等，否则会抛出 ValueError。

列表/元组解包
coordinates = [10, 20]
x, y = coordinates
print(x, y)  # 输出: 10 20

字符串解包
a, b, c = "ABC"
print(a, b, c)  # 输出: A B C

字典解包（默认解包键）
data = {"name": "Alice", "age": 25}
key1, key2 = data
print(key1, key2)  # 输出: name age

✨ 星号解包

当元素数量不确定，或者你只关心其中一部分时，使用 * 运算符。它会将剩余的所有元素收集为一个列表。

收集剩余元素
numbers = [1, 2, 3, 4, 5]
first, *middle, last = numbers
print(first)   # 1
print(middle)  # [2, 3, 4]
print(last)    # 5

仅获取首尾
head, *tail = [10, 20, 30, 40]
print(head)  # 10
print(tail)  # [20, 30, 40]

仅获取最后一个
*head, tail = [10, 20, 30, 40]
print(head)  # [10, 20, 30]
print(tail)  # 40

⚠️ 注意：在一个解包表达式中，* 只能出现一次。

📖 字典解包

字典解包有几种特殊且强大的用法，尤其在函数传参和合并字典时非常关键。

解包键值对
使用 .items() 方法可以同时解包键和值，这在遍历字典时比手动索引优雅得多。

user = {"name": "Bob", "score": 95}
for key, value in user.items():
    print(f"{key}: {value}")

函数参数解包
dict：将字典解包为关键字参数**。
*list：将列表/元组解包为位置参数。

def greet(name, age):
    print(f"Hello {name}, you are {age} years old.")

字典解包为关键字参数
info = {"name": "Charlie", "age": 30}
greet(**info)  # 等价于 greet(name="Charlie", age=30)

列表解包为位置参数
args = ["David", 28]
greet(*args)   # 等价于 greet("David", 28)

合并字典
在 Python 3.9+ 中，| 运算符和 |= 运算符可以直接合并字典，底层原理就是字典解包。

dict1 = {"a": 1, "b": 2}
dict2 = {"b": 3, "c": 4}

合并（dict2 的值会覆盖 dict1 中相同的键）
merged = dict1 | dict2
print(merged)  # {'a': 1, 'b': 3, 'c': 4}

🛠️ 实战应用场景

交换变量
无需临时变量，一行代码完成交换。
a, b = 10, 20
a, b = b, a  # a=20, b=10

函数返回多值
函数返回元组，调用时直接解包接收。
def get_user_info():
    return "Alice", 25, "alice@example.com"

name, age, email = get_user_info()

批量处理多模态文件
结合你之前问的多模态 API，解包可以优雅地处理批量任务。
import os

假设 images 目录下有多个图片文件
image_files = [f for f in os.listdir("./images") if f.endswith(".jpg")]

解包获取前3张图片路径，其余放入剩余列表
first, second, third, *rest = image_files

分别处理
process_image(first)
process_image(second)
process_image(third)

批量处理剩余图片
for img in rest:
    process_image(img)

⚠️ 常见陷阱

数量不匹配：基础解包时变量数与元素数不等会报错，不确定数量时务必使用 *。
*** 只能用一个**：a, *b, *c, d = [1,2,3,4,5] 是非法语法。
空序列解包：对空列表解包会报错，建议先判断长度或使用 * 收集。
字典解包默认是键：直接 k, v = {"a": 1} 会报错，必须用 .items() 或指定 .keys()/.values()。
