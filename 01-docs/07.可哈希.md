简单来说，可哈希就是指一个对象拥有一个固定不变的哈希值，并且支持通过 hash() 函数获取这个值。

在 Python 中，这直接决定了该对象能否被放入字典的键或集合中。

🔍 核心条件

一个对象要成为“可哈希”的，必须同时满足：

实现了 __hash__() 方法：返回一个整数。
实现了 __eq__() 方法：用于判断相等性。
哈希值在生命周期内恒定不变：如果两个对象相等（a == b），它们的哈希值必须相同（hash(a) == hash(b)）。

⚖️ 为什么字典和集合要求“可哈希”？

字典和集合底层使用哈希表实现。查找过程如下：

key → hash(key) → 计算桶位置 → 在该桶内用 == 精确匹配

如果哈希值会变，就会出现灾难性后果：

存入时哈希值为 100，放入桶 100
对象内容改变，哈希值变为 200
查找时去桶 200 找，但对象还在桶 100 → 永远找不到

所以 Python 强制要求：作为字典键或集合元素的对象，必须是不可变的（或至少哈希值不变）。

✅ 可哈希 vs ❌ 不可哈希
类型   是否可哈希   原因
int, float, str, bool   ✅   值不可变

tuple   ✅   元素不可变（且元素本身也可哈希）

frozenset   ✅   不可变集合

list   ❌   可变，哈希值会变

dict   ❌   可变

set   ❌   可变

自定义类（默认）   ✅   默认基于 id() 哈希

自定义类（重写 eq 但未重写 hash）   ❌   Python 自动将 hash 设为 None

⚠️ 常见陷阱

tuple 不一定可哈希

hash((1, 2, 3))        # ✅ 正常
hash((1, [2, 3]))      # ❌ TypeError: unhashable type: 'list'

tuple 的哈希值依赖于其元素，如果元素本身不可哈希，tuple 也不可哈希。

自定义类的默认行为

class Point:
    def init(self, x, y):
        self.x = x
        self.y = y

p1 = Point(1, 2)
p2 = Point(1, 2)

print(p1 == p2)    # False（默认比较 id）
print(hash(p1))    # 某个整数（基于 id）

默认情况下，自定义类是可哈希的（基于对象身份 id），但逻辑上相等的两个对象哈希值不同，这通常不是你想要的。

重写 eq 后 hash 自动失效

class Point:
    def init(self, x, y):
        self.x = x
        self.y = y

    def eq(self, other):
        return self.x == other.x and self.y == other.y

    # 没有重写 hash

p = Point(1, 2)
hash(p)  # ❌ TypeError: unhashable type: 'Point'

Python 的规则：如果你重写了 __eq__ 但没有重写 __hash__，Python 会将 __hash__ 设为 None，使对象不可哈希。这是为了防止逻辑相等但哈希值不同的对象被错误地放入字典/集合中。

正确做法：

class Point:
    def init(self, x, y):
        self.x = x
        self.y = y

    def eq(self, other):
        if not isinstance(other, Point):
            return NotImplemented
        return self.x == other.x and self.y == other.y

    def hash(self):
        return hash((self.x, self.y))  # 委托给 tuple

📌 一句话总结

可哈希 = 哈希值恒定不变 = 可以作为字典键或集合元素。frozenset 之所以可哈希，正是因为它冻结了内容，保证了哈希值在创建后永远不会改变。
