range() 是 Python 中生成整数序列的内置函数，也是 for 循环的最佳搭档。它采用惰性求值机制，不会一次性在内存中生成所有数字，而是按需产生，因此处理百万级循环时依然高效。

📝 三种调用方式

range() 支持 1 到 3 个参数，分别对应不同的生成规则：

单参数：range(stop)
含义：从 0 开始，到 stop 结束（不包含 stop），步长默认为 1。
示例：
        for i in range(5):
        print(i)  # 输出: 0, 1, 2, 3, 4
    

双参数：range(start, stop)
含义：从 start 开始，到 stop 结束（不包含 stop），步长默认为 1。
示例：
        for i in range(2, 6):
        print(i)  # 输出: 2, 3, 4, 5
    

三参数：range(start, stop, step)
含义：从 start 开始，到 stop 结束（不包含 stop），每次递增 step。
示例：
        # 正步长
    for i in range(1, 10, 2):
        print(i)  # 输出: 1, 3, 5, 7, 9

    # 负步长（倒序）
    for i in range(10, 0, -2):
        print(i)  # 输出: 10, 8, 6, 4, 2
    

⚠️ 核心注意事项

左闭右开区间：range(1, 5) 生成的是 1, 2, 3, 4，永远不包含结束值 5。这是初学者最容易踩坑的地方。
步长不能为 0：step 参数为 0 会抛出 ValueError。
参数必须为整数：range() 不支持浮点数。如果需要生成小数序列，可以使用 numpy.arange() 或列表推导式。
惰性求值：range(1000000) 不会占用 100 万个整数的内存，它只是一个轻量级对象，仅在迭代时计算下一个值。

🛠️ 与 for 循环的实战结合

替代传统索引遍历
当需要同时获取索引和元素时，优先使用 enumerate()，而不是 range(len(list))，代码更 Pythonic：
fruits = ["苹果", "香蕉", "橙子"]

❌ 不推荐
for i in range(len(fruits)):
    print(f"{i}: {fruits[i]}")

✅ 推荐
for i, fruit in enumerate(fruits):
    print(f"{i}: {fruit}")

生成固定次数的循环
重复执行 3 次
for _ in range(3):
    print("执行任务")

提示：当循环变量本身不需要使用时，用 _ 作为占位符是 Python 的约定俗成。

💡 常见误区澄清

range() 返回的是 range 对象，不是列表：在 Python 3 中，range(5) 返回 <class 'range'>，如需转为列表需显式调用 list(range(5))。
range() 不支持浮点数：range(0, 1, 0.1) 会报错，如需小数序列请使用 numpy.arange(0, 1, 0.1) 或 [i/10 for i in range(0, 10)]。
range() 的步长可以是负数：但必须确保 start > stop，否则生成的序列为空。例如 range(5, 1, -1) 生成 5, 4, 3, 2，而 range(1, 5, -1) 生成空序列。
